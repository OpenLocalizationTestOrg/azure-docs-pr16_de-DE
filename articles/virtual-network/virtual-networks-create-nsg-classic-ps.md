## <a name="how-to-create-the-nsg-for-the-front-end-subnet"></a>So erstellen Sie die NSG für das front-End-Subnetz
Zum Erstellen einer benannten **NSG-Front-End-** basierend auf dem oben genannten Szenario benannte NSG führen Sie die folgenden Schritte aus:

1. Wenn Sie noch keine Erfahrung Azure PowerShell haben, finden Sie unter [Installieren und Konfigurieren von Azure PowerShell](../powershell-install-configure.md) , und folgen Sie den Anweisungen, ganz nach Ende melden Sie sich bei Azure, und wählen Sie Ihr Abonnement.

3. Erstellen einer benannten **NSG-Front-End-**Netzwerk-Sicherheitsgruppe an.

        New-AzureNetworkSecurityGroup -Name "NSG-FrontEnd" -Location uswest `
            -Label "Front end subnet NSG"

    Erwartetes Ergebnis:

        Name         Location   Label               
        ----         --------   -----               
        NSG-FrontEnd West US    Front end subnet NSG


4. Erstellen einer Sicherheitsregel wird von Access aus dem Internet zu Port 3389 an.

        Get-AzureNetworkSecurityGroup -Name "NSG-FrontEnd" `
        | Set-AzureNetworkSecurityRule -Name rdp-rule `
            -Action Allow -Protocol TCP -Type Inbound -Priority 100 `
            -SourceAddressPrefix Internet  -SourcePortRange '*' `
            -DestinationAddressPrefix '*' -DestinationPortRange '3389'

    Erwartetes Ergebnis:

        Name     : NSG-FrontEnd
        Location : Central US
        Label    : Front end subnet NSG
        Rules    :

                      Type: Outbound

                                                           Prefix          Range         Address Prefix   Port Range

                   ALLOW INTERNET       65001     Allow    *               *             INTERNET         *              *

4. Erstellen einer Sicherheitsregel wird von Access aus dem Internet zu Port 80.

        | Set-AzureNetworkSecurityRule -Name web-rule `

## <a name="how-to-create-the-nsg-for-the-back-end-subnet"></a>So erstellen Sie die NSG für die Back-End-Subnetz
3. Erstellen einer benannten **NSG-Back-End-**Netzwerk-Sicherheitsgruppe an.

        New-AzureNetworkSecurityGroup -Name "NSG-BackEnd" -Location uswest `
            -Label "Back end subnet NSG"

    Erwartetes Ergebnis:

        Name        Location   Label              
        ----        --------   -----              
        NSG-BackEnd West US    Back end subnet NSG


4. Erstellen einer Sicherheitsregel zulassen des Zugriffs aus dem front-End-Subnetz Port 1433 (von SQL Server verwendete Standardport).

        Get-AzureNetworkSecurityGroup -Name "NSG-FrontEnd" `
        | Set-AzureNetworkSecurityRule -Name rdp-rule `
            -Action Allow -Protocol TCP -Type Inbound -Priority 100 `
            -SourceAddressPrefix 192.168.1.0/24  -SourcePortRange '*' `
            -DestinationAddressPrefix '*' -DestinationPortRange '1433'

    Erwartetes Ergebnis:

        Name     : NSG-BackEnd
        Location : Central US
        Label    : Back end subnet NSG
        Rules    :

                   ALLOW VNET INBOUND   65000     Allow    VIRTUAL_NETWORK *             VIRTUAL_NETWORK  *              *

                   ALLOW INTERNET       65001     Allow    *               *             INTERNET         *              *

4. Erstellen einer Sicherheitsregel Access aus dem Subnetz mit dem Internet blockieren.

        Get-AzureNetworkSecurityGroup -Name "NSG-BackEnd" `
        | Set-AzureNetworkSecurityRule -Name block-internet `

            -SourceAddressPrefix '*'  -SourcePortRange '*' `
